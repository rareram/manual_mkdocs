# 데이터 변경 및 확인

테이블이 준비되었으니, 실제 데이터를 추가(INSERT)하고, 변경(UPDATE)하며, 잘 적용되었는지 확인(SELECT)하는 방법을 알아봅니다.

## 1. 데이터 추가 (INSERT)

`products` 테이블에 샘플 상품 데이터를 추가합니다.

```sql
INSERT INTO products (name, description, price) VALUES
('키보드', '기계식 게이밍 키보드', 120.00),
('마우스', '인체공학 무선 마우스', 75.50);
```

## 2. 데이터 확인 (SELECT)

`SELECT` 쿼리를 사용하여 테이블에 저장된 모든 데이터를 조회합니다.

```sql
SELECT * FROM products;
```

다음과 같이 방금 추가한 2개의 레코드가 화면에 출력됩니다.

```
 id |  name  |      description      | price  |          created_at
----+--------+-----------------------+--------+-------------------------------
  1 | 키보드 | 기계식 게이밍 키보드  | 120.00 | 2025-12-24 10:30:00.123456+09
  2 | 마우스 | 인체공학 무선 마우스  |  75.50 | 2025-12-24 10:30:00.123456+09
(2 rows)
```

이 단계까지 성공했다면, PostgreSQL에서 데이터가 변경될 때마다 `pgcdc` 솔루션이 이를 감지할 기본 준비가 완료된 것입니다.

## 3. 데이터 변경 (UPDATE)

데이터 변경을 테스트하기 위해 `id`가 1인 상품의 가격을 수정해 보겠습니다.

```sql
UPDATE products
SET price = 115.00
WHERE id = 1;
```

다시 `SELECT` 쿼리로 확인하면 가격이 변경된 것을 볼 수 있습니다.

```sql
SELECT * FROM products WHERE id = 1;
```

```
 id |  name  |      description      | price  |          created_at
----+--------+-----------------------+--------+-------------------------------
  1 | 키보드 | 기계식 게이밍 키보드  | 115.00 | 2025-12-24 10:30:00.123456+09
(1 row)
```
